DAY 88 - GIT BRANCHING & MERGING COMMANDS

  --------------------------------------------------
  1. Check current commit history
  --------------------------------------------------
  git log –oneline –graph –all

  --------------------------------------------------

2. Create a new branch

git branch feature-greeting

  --------------------------------------------------
  3. Switch to the new branch
  --------------------------------------------------
  git switch feature-greeting # (Older Git version)
  # git checkout feature-greeting

  --------------------------------------------------

4. Make changes to your file (app.py)

Edit app.py manually, then stage and commit:

git add app.py git commit -m “Modified greeting on feature branch”

  --------------------------------------------------
  5. Switch back to main branch
  --------------------------------------------------
  git switch main # (If your default branch is
  master) # git switch master

  --------------------------------------------------

6. Modify the same file on main branch

Edit app.py again, then:

git add app.py git commit -m “Changed greeting on main branch”

  --------------------------------------------------
  7. Merge feature branch into main
  --------------------------------------------------
  git merge feature-greeting

  --------------------------------------------------

8. If merge conflict occurs

Open the conflicted file and resolve manually.

Remove conflict markers (<<<<<<<, =======, >>>>>>>)

git add app.py git commit -m “Resolved merge conflict”

  ----------------------------
  9. View final commit graph
  ----------------------------

git log –oneline –graph –all
